CloneBnB

This app is a clone of the popular accommodation booking site, AirBnB. After registration, users become a 'traveler' and can view/book properties in a number of different cities. Travelers can use the internal messaging service to contact hosts with questions before booking a property. After their stay has been confirmed and completed, users can leave reviews about places that they have stayed. They also have the opportunity to register as a 'host' and list their their own property for booking.


This was a four-person team project that uses a Ruby 2.4/Rails 5 backend. We utilized the Active Model Serializer gem to serialize an internal API that provides administrators with business intelligence data via ActiveRecord and SQL calls. The Faker gem was used to generate ~5000 records in the PostgreSQL database. It makes use of the BCrypt gem for authentication and authorization as well as Twilio to receive an SMS in the case of forgotten passwords. This is a multi-tenancy app the gives the user priveleges based on 4 different roles (unautheticated, traveler, host, and administrator) and permissions were 'handrolled.' It was completed in a two-week sprint in April of 2017.

APICurious

APICurious is a personalized GitHub interface that provides users' with highlights reel of their GitHub activity without any of that pesky code. User's can access a list of their lastest commits, repositories, organizations, followers, people following them, as well as all GitHub activity for them. There's also a link to the real GitHub in case you feel like doing some actual work.


This was a solo project that uses a Ruby 2.4/Rails 5 backend and the Faraday gem to hit GitHub's API. It makes use of GitHub's OAuth service to log in and provide a user token that's attached to query parameters. A PostgreSQL database was used to store user data. It was completed in a four-day sprint in March of 2017.

URLockbox

The URLockbox is an app for tracking your favorite links. It utilizes BCrypt which requires users to create an account with their email address and a password in order to keep track of their links. Once signed in, users must enter a valid URL (https://www.google.com rather than google.com) and a user-chosen title (cannot be blank) to save. Links have an 'unread' status by default but can be changed using the 'Mark as Read/Mark as Unread' buttons. Users can filter their links in one of two ways. The first is to enter text into the filter box, as a user types, their results list will shorten with each key typed. The second is to click on the "Only Read Links" or "Only Unread Links" buttons which limit results to the read status of each link. This app also works in conjunction with HotReads, which tracks the most popular links across all users in the database.


This was a solo project that uses a Ruby 2.4/Rails 5 backend and ansynchronous JavaScript to provide a seemless user experience without needing to reload. It makes use of the BCrypt gem for authentication and authorization as well as the validate_url gem to ensure the validity of web addresses. The sister app (HotReads) also employs the rack_cors gem in order to overcome the single origin policy during AJAX calls. It was completed in a two-day sprint in June of 2017.
